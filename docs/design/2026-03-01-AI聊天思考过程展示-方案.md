# AI聊天思考过程展示 - 设计方案

## 1. 需求概述

在SCM智能助手的聊天中，当用户发送问题触发工作流执行时，用户当前只能看到"SCM正在思考..."的加载动画，无法了解AI正在做什么。

本方案在工作流执行期间，向用户展示**实时的思考步骤列表**（节点级粒度），完成后自动消失，通过底部标签可回看执行详情。

## 2. 参考项目

| 项目 | 方案特点 | 评估 |
|------|---------|------|
| FastGPT | 16种SSE事件、实时状态气泡、工具手风琴、引用标签、完整响应弹窗 | 功能全面，但前端工作量大 |
| Spring AI Alibaba DeepResearch | ThoughtChain组件、3步宏观进度、完成后折叠 | 粒度太粗，无节点级展示 |
| **本方案** | 节点级步骤列表、完成后消失、底部标签回看、可展开详情 | 取两者之长，适合SCM场景 |

## 3. 交互设计

### 3.1 三阶段时序

#### 阶段1：执行中 — 步骤列表实时展开

```
┌─ AI 消息气泡 ────────────────────────────┐
│                                          │
│  ✓ 意图识别     采购咨询           0.3s  │
│  ✓ 知识库检索   匹配3条            1.2s  │
│  ⏳ 生成回答    正在生成...               │
│                                          │
│  ─────────────────────────────────────── │
│  采购订单的创建步骤如下...                │
│                                          │
└──────────────────────────────────────────┘
```

- 每个节点开始执行时，新增一行 `⏳ {节点名称} {动作}...`
- 节点完成后，变为 `✓ {节点名称} {摘要} {耗时}`
- LLM流式文本在步骤列表下方同步输出（分割线隔开）

#### 阶段2：完成 — 步骤列表直接消失

```
┌─ AI 消息气泡 ────────────────────────────┐
│                                          │
│  采购订单的创建步骤如下：                  │
│  1. 进入采购管理模块                      │
│  2. 点击新建按钮，填写供应商...            │
│  3. 提交审批                              │
│                                          │
│  ┌─────────┐ ┌───────┐ ┌─────────┐     │
│  │📚 引用 3条│ │⏱ 2.1s │ │📄 查看详情│     │
│  └─────────┘ └───────┘ └─────────┘     │
└──────────────────────────────────────────┘
```

- 工作流完成（收到isComplete=true）后，步骤列表立即消失
- 底部出现元信息标签行

#### 阶段3：回看 — 点击"查看详情"展开

```
┌─ AI 消息气泡 ────────────────────────────┐
│                                          │
│  采购订单的创建步骤如下：...               │
│                                          │
│  ┌─ 执行详情 ────────────────────────┐   │
│  │                                    │   │
│  │  ✓ 意图识别     采购咨询     0.3s  │   │
│  │                                    │   │
│  │  ▶ 知识库检索   匹配3条      1.2s  │   │
│  │  ┌───────────────────────────┐    │   │
│  │  │ 📄 采购流程规范.pdf          │    │   │
│  │  │    采购订单审批流程需要...    │    │   │
│  │  │ 📄 供应商管理手册.docx       │    │   │
│  │  │    供应商评级标准分为...      │    │   │
│  │  └───────────────────────────┘    │   │
│  │                                    │   │
│  │  ✓ LLM回答     deepseek     0.6s  │   │
│  │                                    │   │
│  └────────────────────────────────────┘   │
│                                          │
│  ┌─────────┐ ┌───────┐ ┌─────────┐     │
│  │📚 引用 3条│ │⏱ 2.1s │ │📄 收起详情│     │
│  └─────────┘ └───────┘ └─────────┘     │
└──────────────────────────────────────────┘
```

- "查看详情"变为"收起详情"
- 每个节点行可点击展开查看具体内容

### 3.2 展示的节点类型（6种）

| 节点类型 | 执行中文本 | 完成摘要 | 展开详情内容 |
|---------|-----------|---------|------------|
| 分类器 | `⏳ 意图识别 分析中...` | `✓ 意图识别 → 采购咨询` | 无需展开 |
| 知识库检索 | `⏳ 知识库检索 检索中...` | `✓ 知识库检索 → 匹配N条` | 每条：文档名 + 内容摘要 |
| 临时知识库 | `⏳ 创建临时知识库...` | `✓ 临时知识库 → 创建完成` | 无需展开 |
| LLM回答 | `⏳ 生成回答 正在生成...` | `✓ 生成回答 → {模型名}` | Token用量 |
| MCP工具 | `⏳ 调用工具 {工具名}...` | `✓ 调用工具 → {工具名}` | 参数JSON + 结果JSON |
| 文档解析 | `⏳ 解析文档...` | `✓ 文档解析 → 提取完成` | 无需展开 |

**不展示的节点**：Start、End、Template、Switcher、SubWorkflow、HttpRequest、MailSend、KeywordExtractor、FaqExtractor

### 3.3 底部标签规则

| 标签 | 出现条件 | 内容 |
|------|---------|------|
| 📚 引用 N条 | 工作流含知识库检索且有结果 | 点击滚动到详情中的知识库部分 |
| 🔧 工具 N个 | 工作流含MCP工具节点 | 点击滚动到详情中的工具部分 |
| ⏱ X.Xs | 始终显示（工作流模式） | 总耗时 |
| 📄 查看详情 | 始终显示（工作流模式） | 切换展开/收起执行详情面板 |

### 3.4 特殊场景

| 场景 | 处理 |
|------|------|
| 普通聊天（无工作流） | 不显示思考过程，保持现有"SCM正在思考..."行为 |
| 工作流只有1个LLM节点 | 只显示 `⏳ 生成回答 正在生成...`，完成后消失 |
| 节点执行失败 | 显示 `✗ {节点名} 失败`（红色） |
| 并行节点 | 按完成顺序依次出现 |
| Orchestrator模式 | 同样展示工作流内的节点步骤 |

## 4. 数据结构设计

### 4.1 新增后端SSE事件（2种）

在现有4种事件（runtime、chunk、output、interrupt）基础上，新增：

#### node_start事件

节点开始执行时发送。

```json
{
  "type": "node_start",
  "node": "uuid-xxx",
  "nodeName": "KnowledgeRetrieval",
  "nodeTitle": "知识库检索",
  "timestamp": 1709280000000
}
```

#### node_complete事件

节点执行完成时发送。

```json
{
  "type": "node_complete",
  "node": "uuid-xxx",
  "nodeName": "KnowledgeRetrieval",
  "nodeTitle": "知识库检索",
  "duration": 1200,
  "summary": {
    "matchCount": 3,
    "items": [
      { "docName": "采购流程规范.pdf", "snippet": "采购订单审批流程需要经过..." },
      { "docName": "供应商管理手册.docx", "snippet": "供应商评级标准分为..." }
    ]
  }
}
```

各节点类型的summary结构：

| 节点类型 | summary字段 |
|---------|------------|
| 分类器 | `{ "result": "采购咨询" }` |
| 知识库检索 | `{ "matchCount": 3, "items": [{ "docName", "snippet" }] }` |
| 临时知识库 | `{ "kbUuid": "..." }` |
| LLM回答 | `{ "model": "deepseek-chat", "promptTokens": 500, "completionTokens": 200 }` |
| MCP工具 | `{ "toolName": "openPage", "params": {...}, "result": {...} }` |
| 文档解析 | `{ "fileCount": 1 }` |

### 4.2 前端消息数据结构扩展

在现有消息对象基础上，新增 `workflowProcess` 字段：

```javascript
{
  // ... 现有字段保持不变 ...

  // 新增：工作流执行过程数据
  workflowProcess: {
    isWorkflow: true,
    totalDuration: 2100,
    showDetail: false,        // 是否展开详情面板
    nodes: [
      {
        nodeUuid: "uuid-1",
        nodeName: "Classifier",
        nodeTitle: "意图识别",
        status: "completed",  // running | completed | error
        duration: 300,
        summary: { result: "采购咨询" },
        detailExpanded: false // 是否展开此节点详情
      },
      {
        nodeUuid: "uuid-2",
        nodeName: "KnowledgeRetrieval",
        nodeTitle: "知识库检索",
        status: "completed",
        duration: 1200,
        summary: { matchCount: 3, items: [...] },
        detailExpanded: false
      }
    ]
  }
}
```

### 4.3 节点展示过滤表

```javascript
const VISIBLE_NODES = {
  'Classifier': { title: '意图识别', runningText: '分析中...' },
  'KnowledgeRetrieval': { title: '知识库检索', runningText: '检索中...' },
  'TempKnowledgeBase': { title: '创建临时知识库', runningText: '创建中...' },
  'Answer': { title: '生成回答', runningText: '正在生成...' },
  'McpTool': { title: '调用工具', runningText: '' },  // 动态显示工具名
  'DocumentExtractor': { title: '文档解析', runningText: '解析中...' }
}
```

## 5. 改动范围

### 5.1 后端改动

| 文件 | 改动 | 工作量 |
|------|------|:---:|
| WorkflowEngine.java | 节点process()前后各发一条事件(node_start/node_complete) | 小 |
| 各节点类（6种） | 在process()返回前构造summary数据 | 中 |
| WorkflowEventVo.java | 新增node_start/node_complete的工厂方法 | 小 |
| WorkflowEventAdapter.java | 新事件类型转ChatResponseVo | 小 |
| ChatResponseVo.java | 新增nodeStatus相关字段 | 小 |

### 5.2 前端改动

| 文件 | 改动 | 工作量 |
|------|------|:---:|
| aiChatService.js | SSE解析新增node_start/node_complete事件处理 | 小 |
| chat.js (store) | 消息数据结构扩展workflowProcess，mutations处理 | 中 |
| MessageList.vue | 新增ThinkingSteps组件渲染 | 中 |
| 新建: ThinkingSteps.vue | 步骤列表组件（执行中状态） | 中 |
| 新建: ExecutionDetail.vue | 执行详情面板组件（回看状态） | 中 |
| 新建: ResponseTags.vue | 底部标签组件 | 小 |

### 5.3 不做的事情（KISS原则）

- 不做节点连线图可视化
- 不做实时Token计数动画
- 不做并行节点的分支展示
- 不做子工作流递归展开
- 不做节点日志实时滚动
