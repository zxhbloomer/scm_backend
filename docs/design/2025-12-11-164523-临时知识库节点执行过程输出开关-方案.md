# 临时知识库节点增加"执行过程输出"开关

## 需求背景

用户在测试工作流时发现，即使上游节点的"执行过程输出"开关关闭，聊天界面仍然显示"OK"。经过排查，发现"OK"来自临时知识库节点（TempKnowledgeBase），该节点目前没有"执行过程输出"开关。

**问题本质**: 临时知识库节点缺少`show_process_output`配置，导致其输出始终显示在聊天界面。

## 需求描述

为临时知识库节点增加"执行过程输出"开关：
- 默认：开启（显示执行过程）
- 关闭后：不在流式输出显示具体过程，但数据仍传递给下游节点

与MCP工具节点、Switcher节点的实现保持一致。

## 技术方案

### 核心判断

✅ **值得做**：这是一个真实的用户痛点，与Switcher节点问题完全相同

### 关键洞察

- **数据结构**：`TempKnowledgeBaseNodeConfig` 需要增加 `showProcessOutput` 字段
- **复杂度**：WorkflowEngine已支持通用的`show_process_output`检查，无需修改
- **风险点**：零破坏性，纯增量改动

### 改动清单

#### 1. 后端：TempKnowledgeBaseNodeConfig.java

**文件路径**: `scm-ai/src/main/java/com/xinyirun/scm/ai/workflow/node/tempknowledgebase/TempKnowledgeBaseNodeConfig.java`

**改动内容**: 增加 `showProcessOutput` 字段

```java
package com.xinyirun.scm.ai.workflow.node.tempknowledgebase;

import com.fasterxml.jackson.annotation.JsonProperty;
import lombok.Data;

@Data
public class TempKnowledgeBaseNodeConfig {

    private String model_name;

    private String brief;

    /**
     * 是否显示执行过程输出到chat流
     * true(默认): 流式输出显示在聊天界面
     * false: 不显示流式输出，但结果仍传递给下游节点
     */
    @JsonProperty("show_process_output")
    private Boolean showProcessOutput = true;
}
```

#### 2. 前端：TempKnowledgeBaseNodeProperty.vue

**文件路径**: `src/components/70_ai/components/workflow/components/properties/TempKnowledgeBaseNodeProperty.vue`

**改动内容**:

**Template部分新增** (在LLM选择器之后):
```vue
<!-- 执行过程输出开关 -->
<div class="property-section">
  <div class="section-title">
    执行过程输出
    <el-tooltip content="关闭后，节点执行结果不会显示在对话中，但仍会传递给下游节点" placement="top">
      <i class="el-icon-question" style="color: #909399; font-size: 14px; margin-left: 4px;" />
    </el-tooltip>
  </div>
  <el-switch
    v-model="nodeConfig.show_process_output"
    active-text="显示"
    inactive-text="隐藏"
  />
</div>
```

**Computed部分修改** (在nodeConfig计算属性中增加初始化):
```javascript
nodeConfig () {
  // ... 现有初始化代码 ...

  // 执行过程输出开关，默认为true（显示）
  if (this.wfNode.nodeConfig.show_process_output === undefined) {
    this.$set(this.wfNode.nodeConfig, 'show_process_output', true)
  }

  return this.wfNode.nodeConfig
}
```

### 不需要改动的文件

- **WorkflowEngine.java**: 已支持通用的 `show_process_output` 检查，无需修改
- **TempKnowledgeBaseNode.java**: 节点逻辑不变，数据仍传递给下游节点

### 数据库

无需改动。`show_process_output` 存储在 `ai_workflow_node.node_config` JSON字段中。

## 测试验证

1. 打开工作流编辑器
2. 选中临时知识库节点
3. 确认右侧属性面板显示"执行过程输出"开关，默认为"显示"
4. 将开关切换为"隐藏"并保存工作流
5. 执行工作流，确认临时知识库节点的输出不再显示在聊天界面
6. 确认下游节点仍能正常接收数据

## 向后兼容性

- 现有工作流的临时知识库节点没有 `show_process_output` 字段
- 前端初始化时会设置默认值为 `true`（显示）
- WorkflowEngine检查逻辑：字段不存在时默认显示
- 完全向后兼容，不影响现有工作流
