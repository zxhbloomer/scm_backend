# 知识检索节点增加"执行过程输出"开关

## 需求背景

用户希望知识检索节点也支持"执行过程输出"开关，与临时知识库节点、Switcher节点、MCP工具节点保持一致。当开关关闭时，节点执行结果不在聊天界面显示，但数据仍传递给下游节点。

## 需求描述

为知识检索节点增加"执行过程输出"开关：
- 默认：开启（显示执行过程）
- 关闭后：不在流式输出显示具体过程，但数据仍传递给下游节点

## 调用链路分析

```
用户点击开关 → KnowledgeRetrievalNodeProperty.vue (前端)
    ↓
nodeConfig.show_process_output = true/false
    ↓
保存工作流 → ai_workflow_node.node_config (JSON字段)
    ↓
执行工作流 → WorkflowEngine.java:executeNodeInternal()
    ↓
KnowledgeRetrievalNode.onProcess() 返回 NodeProcessResult
    ↓
WorkflowEngine 检查 nodeConfig.show_process_output
    ↓
如果 false → 跳过 NODE_OUTPUT SSE事件
如果 true  → 发送 NODE_OUTPUT SSE事件到前端
```

**关键差异：**
- 知识检索节点**不主动调用**`sendNodeChunk()`
- 由WorkflowEngine在第532-549行统一检查`show_process_output`配置，控制`NODE_OUTPUT`事件

## 技术方案

### 核心判断

✅ **值得做**：用户明确需求，与其他节点保持一致

### 关键洞察

- **数据结构**：`KnowledgeRetrievalNodeConfig` 需要增加 `showProcessOutput` 字段
- **复杂度**：WorkflowEngine已有通用检查逻辑，后端节点类无需修改
- **风险点**：零破坏性，纯增量改动

### 改动清单

#### 1. 后端：KnowledgeRetrievalNodeConfig.java

**文件路径**: `scm-ai/src/main/java/com/xinyirun/scm/ai/workflow/node/knowledgeretrieval/KnowledgeRetrievalNodeConfig.java`

**改动内容**: 增加 `showProcessOutput` 字段

```java
/**
 * 是否显示执行过程输出到chat流
 * true(默认): 流式输出显示在聊天界面
 * false: 不显示流式输出，但结果仍传递给下游节点
 */
@JsonProperty("show_process_output")
private Boolean showProcessOutput = true;
```

#### 2. 前端：KnowledgeRetrievalNodeProperty.vue

**文件路径**: `src/components/70_ai/components/workflow/components/properties/KnowledgeRetrievalNodeProperty.vue`

**Template部分新增** (在"默认回复内容"之后):
```vue
<!-- 执行过程输出开关 -->
<div class="property-section">
  <div class="section-title">
    执行过程输出
    <el-tooltip content="关闭后，节点执行结果不会显示在对话中，但仍会传递给下游节点" placement="top">
      <i class="el-icon-question" style="color: #909399; font-size: 14px; margin-left: 4px;" />
    </el-tooltip>
  </div>
  <el-switch
    v-model="nodeConfig.show_process_output"
    active-text="显示"
    inactive-text="隐藏"
  />
</div>
```

**Computed部分修改** (在nodeConfig计算属性中增加初始化):
```javascript
// 执行过程输出开关，默认为true（显示）
if (this.wfNode.nodeConfig.show_process_output === undefined) {
  this.$set(this.wfNode.nodeConfig, 'show_process_output', true)
}
```

### 不需要改动的文件

- **WorkflowEngine.java**: 已有通用检查逻辑（第532-549行），自动支持新字段
- **KnowledgeRetrievalNode.java**: 不主动发送流式输出，无需修改

### 数据库

无需改动。`show_process_output` 存储在 `ai_workflow_node.node_config` JSON字段中。

## 测试验证

1. 打开工作流编辑器
2. 选中知识检索节点
3. 确认右侧属性面板显示"执行过程输出"开关，默认为"显示"
4. 将开关切换为"隐藏"并保存工作流
5. 执行工作流，确认知识检索节点的输出不再显示在聊天界面
6. 确认下游节点仍能正常接收数据

## 向后兼容性

- 现有工作流的知识检索节点没有 `show_process_output` 字段
- 前端初始化时会设置默认值为 `true`（显示）
- WorkflowEngine检查逻辑：字段不存在时默认显示
- 完全向后兼容，不影响现有工作流
