# Switcher多目标并行执行方案

## 一、完整调用链路

```
AI Chat / Workflow测试
  └─ WorkflowEngine.init()
       └─ buildStateGraph(stateGraph, startNode)
            ├─ [步骤1] addNodeToStateGraph(stateGraph, uuid)  -- 注册所有节点
            ├─ [步骤2] addEdge(START, startNode)
            ├─ [步骤3] 收集条件分支节点UUID (Switcher/Classifier)
            ├─ [步骤4] addEdgeToStateGraph(source, target)  -- 普通边（跳过Switcher出边）
            ├─ [步骤5] processConditionalEdges(stateGraph)  ← 问题所在
            └─ [步骤6] addEdge(endNode, END)

processConditionalEdges(stateGraph)
  └─ 按sourceHandle分组Switcher出边
       ├─ uniqueTargets.size() == 1 → mappings.put(handle, target)  [正常]
       └─ uniqueTargets.size() > 1  → 随机取一个，丢弃其余        [BUG]
            └─ stateGraph.addConditionalEdges(switcherUuid, condition, mappings)
```

```
开始

            ├─ [pZkTvFMA] → 临时知识库 → 临时知识库检索 → 大模型 → 结束1
```

## 二、问题诊断与根因

### 根因：WorkflowEngine.java:923-930

```java

    // BUG：HashSet.iterator().next() 无顺序保证，随机取一个
    String targetUuid = uniqueTargets.iterator().next();
```

**问题表现**：
- pZkTvFMA handle 在数据库中有 2 条出边（临时知识库 + 项目管理知识库检索）
- 代码只保留其中 1 个，另一个被静默丢弃
- 由于 HashSet 无顺序，每次重启 JVM 可能丢弃不同的目标

**数据证据**（workflow_id=34 clean数据，is_deleted=0）：
```
edge id=194: pZkTvFMA → 临时知识库
edge id=188: pZkTvFMA → 项目管理知识库检索   ← 被丢弃
```

## 三、方案设计

### 核心思路（参考 Spring AI Alibaba ParallelNode 模式）

当 sourceHandle 有多个目标时：
1. 找到所有分支链的汇聚点（大模型）
2. 为每条分支链追踪有序节点列表
3. 创建虚拟并行节点，内部用 CompletableFuture 并行执行所有链
4. 映射 handle → 虚拟并行节点，虚拟并行节点 → 汇聚点

执行拓扑变化：
```
修复前：Switcher →[pZkTvFMA]→ 随机一个节点

修复后：Switcher →[pZkTvFMA]→ virtual_parallel_XXXX_pZkTvFMA
                                    ├─ CompletableFuture: 临时知识库 → 临时知识库检索
                                    └─ CompletableFuture: 项目管理知识库检索
                                    ↓（allOf完成后）
                               大模型（汇聚点）→ 结束1
```

### 单目标 handle 完全不变

handle → 单目标的路径（如 default → 大模型1）逻辑完全不动。

```java
} else {
    // 多目标：创建虚拟并行节点并行执行所有分支链
    String convergenceNodeId = findConvergenceNodeUuid(uniqueTargets);
    if (convergenceNodeId == null) {
        // 找不到汇聚点，退化为取第一个（兜底）
        String targetUuid = new ArrayList<>(uniqueTargets).get(0);
        mappings.put(sourceHandle, targetUuid);
        log.warn("[processConditionalEdges] sourceHandle {} 找不到汇聚点，退化为单目标: {}",
                sourceHandle, targetUuid);
        continue;
    }

    // 追踪每条分支链（从各目标到汇聚点，不含汇聚点）
    List<List<String>> chains = new ArrayList<>();
    for (String targetId : uniqueTargets) {
        chains.add(traceChainNodeUuids(targetId, convergenceNodeId));
    }

    // 创建虚拟并行节点
    String virtualNodeId = WorkflowConstants.VIRTUAL_PARALLEL_PREFIX
            + switcherUuid + "_" + sourceHandle;

    stateGraph.addNode(virtualNodeId, state -> CompletableFuture.supplyAsync(() ->
            executeParallelChains(chains, state)));

    mappings.put(sourceHandle, virtualNodeId);
    addEdgeToStateGraph(stateGraph, virtualNodeId, convergenceNodeId);
    log.info("[processConditionalEdges] sourceHandle {} -> 并行节点 {}, 汇聚点 {}, 分支数 {}",
            sourceHandle, virtualNodeId, convergenceNodeId, chains.size());
}
```

#### 修改2：新增私有方法 findConvergenceNodeUuid()

```java
/**
 * 找到多条分支链的汇聚点。
 * 依次追踪每条链的下游节点，返回第一个被所有链共同到达的节点。
 */
private String findConvergenceNodeUuid(Set<String> startNodeIds) {
    List<List<String>> allChainNodes = new ArrayList<>();
    for (String startId : startNodeIds) {
        List<String> downstream = new ArrayList<>();
        String current = startId;
        Set<String> visited = new HashSet<>();
        while (current != null && !visited.contains(current)) {
            downstream.add(current);
            visited.add(current);
            current = getDirectNextNodeUuid(current);
        }
        allChainNodes.add(downstream);
    }

    // 取第一条链的节点集，与其他链求交集
    Set<String> commonNodes = new LinkedHashSet<>(allChainNodes.get(0));
    for (int i = 1; i < allChainNodes.size(); i++) {
        commonNodes.retainAll(new HashSet<>(allChainNodes.get(i)));
    }
    if (commonNodes.isEmpty()) {
        return null;
    }
    // 返回在第一条链中最早出现的公共节点
    for (String node : allChainNodes.get(0)) {
        if (commonNodes.contains(node)) {
            return node;
        }
    }
    return null;
}
```

#### 修改3：新增私有方法 traceChainNodeUuids()

```java
/**
 * 追踪从 startNodeId 到 convergenceNodeId 的有序节点列表（不含汇聚点）。
 */
private List<String> traceChainNodeUuids(String startNodeId, String convergenceNodeId) {
    List<String> chain = new ArrayList<>();
    String current = startNodeId;
    Set<String> visited = new HashSet<>();
    while (current != null
            && !current.equals(convergenceNodeId)
            && !visited.contains(current)) {
        chain.add(current);
        visited.add(current);
        current = getDirectNextNodeUuid(current);
    }
    return chain;
}
```

#### 修改4：新增私有方法 getDirectNextNodeUuid()

```java
/**
 * 获取节点的直接后继节点UUID（仅适用于单出边的普通节点）。
 */
private String getDirectNextNodeUuid(String nodeUuid) {
    return wfEdges.stream()
            .filter(e -> nodeUuid.equals(e.getSourceNodeUuid()))
            .map(AiWorkflowEdgeEntity::getTargetNodeUuid)
            .findFirst()
            .orElse(null);
}
```

#### 修改5：新增私有方法 executeParallelChains()

```java
/**
 * 并行执行多条节点链，合并所有链的输出结果。
 * 参考 Spring AI Alibaba ParallelNode 的 CompletableFuture + 状态快照 + 合并 模式。
 */
private Map<String, Object> executeParallelChains(List<List<String>> chains, OverAllState parentState) {
    // 对父图状态做快照，各分支独立使用（防止竞态写冲突）
    Map<String, Object> snapshotData = new HashMap<>(parentState.data());

    List<CompletableFuture<Map<String, Object>>> futures = chains.stream()
            .map(chain -> CompletableFuture.supplyAsync(() -> {
                DataSourceHelper.use(this.tenantCode);
                Map<String, Object> chainState = new HashMap<>(snapshotData);
                Map<String, Object> chainResult = new HashMap<>();
                for (String nodeUuid : chain) {
                    AiWorkflowNodeVo node = getNodeByUuid(nodeUuid);
                    WfNodeState nodeState = new WfNodeState(chainState);
                    Map<String, Object> nodeResult = runNode(node, nodeState);
                    // 链内状态向下传递
                    chainState.putAll(nodeResult);
                    chainResult.putAll(nodeResult);
                }
                return chainResult;
            }))
            .collect(Collectors.toList());

    CompletableFuture.allOf(futures.toArray(new CompletableFuture[0])).join();

    Map<String, Object> merged = new HashMap<>();
    for (CompletableFuture<Map<String, Object>> future : futures) {
        try {
            merged.putAll(future.get());
        } catch (Exception e) {
            throw new RuntimeException("并行分支执行失败", e);
        }
    }
    return merged;
}
```

## 五、KISS 原则 7 问题

1. **这是个真问题还是臆想？** 真问题。workflow_id=34 数据库中 pZkTvFMA 明确有 2 条出边，代码丢弃其中 1 条，已确认。

2. **有更简单的方法吗？** 是当前最简。只改 WorkflowEngine.java 一个文件，新增 4 个私有方法约 80 行代码，复用现有 runNode()。

3. **会破坏什么吗？** 不会。单目标 handle 逻辑完全不变（只有 uniqueTargets.size() > 1 才走新分支）。所有现有工作流不受影响。

4. **当前项目真的需要这个功能吗？** 是。采购-项目管理-新增工作流（workflow_id=34）已经画好了并行分支，需要修复才能正确执行。

5. **过度设计了吗？** 没有。不引入新类，不用子图编译，不引入复杂框架机制。直接调用 runNode()，用 CompletableFuture 并行，用 putAll 合并。

6. **话题是否模糊，可能幻觉？** 不会。问题定位明确（WorkflowEngine.java:923-930），解法参考 Spring AI Alibaba ParallelNode 成熟模式。

7. **是否学习了代码实施注意事项？** 是。注意事项第12条（租户在异步线程中需要 DataSourceHelper.use）、代码复用 runNode()、不新增多余抽象。

## 六、风险分析与缓解

| 风险 | 可能性 | 缓解措施 |
|------|--------|--------|
| 分支链内有 Switcher 节点（嵌套条件） | 低 | getDirectNextNodeUuid 只取第一条出边，Switcher 子图复杂情况暂不处理，兜底日志告警 |
| 汇聚点找不到（两条链不收敛） | 低 | convergenceNodeId == null 时退化为单目标，保留现有行为 |
| 并行分支写入同一 state key 冲突 | 低 | 各分支输出 key 基于 NODE_OUTPUT_KEY_PREFIX + nodeUuid，不同节点不冲突 |
| compile() 拒绝孤立节点 | 低 | 分支链节点虽孤立但已注册，Spring AI Alibaba 不校验可达性 |
| 异步线程丢失租户上下文 | 已处理 | executeParallelChains 首行调用 DataSourceHelper.use(this.tenantCode) |
