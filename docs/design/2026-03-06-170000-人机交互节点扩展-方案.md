# 人机交互节点扩展 - 方案设计文档

> 日期: 2026-03-06
> 功能: 工作流人机交互节点支持4种交互类型
> 状态: 待审批

## 1. 需求概述

扩展工作流 HumanFeedbackNode 节点，从当前仅支持 tip 文本提示，扩展为支持 4 种交互类型（自由文本 / 确认驳回 / 单项选择 / 表单填写），并打通前后端闭环。

## 2. 完整调用链路

```
用户在工作流编辑器配置人机交互节点
        │
        ▼
nodeConfig (JSON) 保存到 ai_workflow_node 表
        │
        ▼ 用户运行工作流
WorkflowStarter.streaming()
  → new WorkflowEngine(... interactionService ...)
  → WorkflowEngine.run()
  → executeWorkflow(false)
    → 图执行到达 HumanFeedbackNode 之前
    → runNode() 检测到 interruptNode → 返回空结果
    → handleInterruption()
      ├─ getHumanFeedbackConfig() 读取节点config
      ├─ buildInteractionParams() 构建交互参数(含动态选项解析)
      ├─ interactionService.createInteraction() → DB记录(WAITING)
      └─ 发送 SSE: WorkflowEventVo.createInterruptDataWithInteraction()
            │
            ▼ SSE事件到达前端
前端 aiChatService.js
  → 解析 chatResponse.interaction_request
  → AiInteractionManager.startInteraction() → Vuex存储 + 倒计时
  → 根据 interaction_type 渲染:
    ├─ AiUserConfirm.vue (confirm类型)
    ├─ AiUserSelect.vue  (select类型)
    ├─ AiUserForm.vue    (form类型)
    └─ 文本输入框         (text类型)
            │
            ▼ 用户操作
AiInteractionManager.submitFeedback()
  → 构建 JSON: {type, interaction_uuid, action, data}
  → 发送到后端 /resume 接口
            │
            ▼
WorkflowRoutingService.resumeWorkflow()
  → WorkflowStarter.resumeFlowAsFlux()
  → interactionService.submitFeedback() → DB更新(SUBMITTED)
  → WorkflowEngine.resume(userInput)
  → app.updateState(HUMAN_FEEDBACK_KEY, userInput)
  → executeWorkflow(true)
  → HumanFeedbackNode.onProcess()
    → parseUserFeedback(userInput)
    → 输出多个 NodeIOData (action / output / selectedKey / formData)
            │
            ▼
下游节点(如条件分支)读取 action 字段进行路由
```

### 超时链路（定时任务）

```
InteractionTimeoutScheduler (@Scheduled 60秒)
  → interactionService.findExpiredInteractions()
  → interactionService.timeoutInteraction() → DB更新(TIMEOUT)
  → InterruptedFlow.RUNTIME_TO_GRAPH.remove() → 终止工作流
```

## 6. 关键技术决策

### 6.1 WorkflowEngine 注入 interactionService

WorkflowEngine 是 new 出来的（由 WorkflowStarter 创建），不是 Spring Bean。
方案：WorkflowStarter 中 @Resource 注入 AiWorkflowInteractionService，通过构造函数传给 WorkflowEngine。

### 6.2 动态选项解析

handleInterruption() 在中断时从 wfState.getCompletedNodes() 中查找上游节点输出，匹配 dynamicOptionsParam 参数名，解析为 SelectOption 列表。

### 6.3 向后兼容

- interactionType 默认 "text"
- 老的 nodeConfig 只有 tip 时，行为与改造前完全一致
- resume 接口仍接受纯文本字符串，HumanFeedbackNode 先尝试 JSON 解析，失败则当纯文本处理

## 7. 风险分析

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| WorkflowEngine构造函数改变影响子工作流 | 低 | 中 | 子工作流构造也传入interactionService |
| SSE interrupt事件前端处理路径不匹配 | 中 | 高 | 需确认前端处理链路，必要时调整 |
| 动态选项JSON格式不规范 | 低 | 低 | 解析失败返回空列表，不影响流程 |
| 超时终止时工作流状态不一致 | 低 | 中 | 超时只移除缓存，不改runtime表状态（已由DB记录TIMEOUT） |
