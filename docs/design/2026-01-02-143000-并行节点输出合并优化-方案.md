# 并行节点输出合并优化方案

## 1. 问题描述

### 1.1 现象
工作流引擎在处理并行节点时，下游节点只能获取到最后一个完成的并行节点的输出，其他并行节点的输出丢失。

### 1.2 影响场景
- 工作流id=34：两个并行知识库检索节点汇聚到一个大模型节点
  - 节点A：项目管理知识库检索（uuid: AbBFx...）
  - 节点B：临时知识库检索（uuid: ejbaB...）
  - 汇聚节点：大模型（uuid: X3HbP...）
- 大模型节点只能获取到节点A或节点B之一的输出

### 1.3 根本原因
`WfState.getLatestOutputs()`方法只取completedNodes列表的最后一个节点输出：
```java
// WfState.java:131-133
public List<NodeIOData> getLatestOutputs() {
    return completedNodes.get(completedNodes.size() - 1).getState().getOutputs();
}
```

## 3. 解决方案

### 3.1 方案核心思想
让节点输出通过OverAllState流动，利用框架自动合并机制，而不是通过completedNodes列表传递。

### 3.2 修改文件清单

| 文件 | 修改类型 | 说明 |
|------|----------|------|
| WorkflowConstants.java | 新增常量 | 节点输出key前缀 |
| WorkflowEngine.java | 修改runNode() | 返回值增加节点输出 |
| AbstractWfNode.java | 修改initInput() | 从OverAllState获取上游输出 |

### 3.3 详细设计

#### 3.3.1 WorkflowConstants.java - 新增常量
```java
/**
 * 节点输出在OverAllState中的key前缀
 * 格式: NODE_OUTPUT_PREFIX + nodeUuid
 */
public static final String NODE_OUTPUT_KEY_PREFIX = "node_output_";
```

#### 3.3.2 WorkflowEngine.java - 修改runNode()返回值

**修改位置**: runNode()方法，约第560-567行

**修改前**:
```java
// 7. 设置节点名称
resultMap.put("name", wfNode.getTitle());
return resultMap;
```

**修改后**:
```java
// 7. 设置节点名称
resultMap.put("name", wfNode.getTitle());

// 8. 将节点输出放入返回Map，让框架自动合并到OverAllState
// 使用nodeUuid作为key确保并行节点输出不会互相覆盖
String outputKey = NODE_OUTPUT_KEY_PREFIX + wfNode.getUuid();
resultMap.put(outputKey, nodeState.getOutputs());

return resultMap;
```

#### 3.3.3 AbstractWfNode.java - 修改initInput()获取方式

**修改位置**: initInput()方法，约第81-87行

**修改前**:
```java
List<NodeIOData> inputs = new ArrayList<>();

// 将上游节点的输出转成当前节点的输入
List<NodeIOData> upstreamOutputs = wfState.getLatestOutputs();
if (!upstreamOutputs.isEmpty()) {
    inputs.addAll(new ArrayList<>(deepCopyList(upstreamOutputs)));
}
```

**修改后**:
```java
List<NodeIOData> inputs = new ArrayList<>();

// 从OverAllState获取所有上游节点的输出
// 参照Spring AI Alibaba方式：通过state.data()获取，框架自动合并并行节点输出
Map<String, Object> stateData = state.data();
for (Map.Entry<String, Object> entry : stateData.entrySet()) {
    String key = entry.getKey();
    if (key.startsWith(NODE_OUTPUT_KEY_PREFIX)) {
        Object value = entry.getValue();
        if (value instanceof List) {
            @SuppressWarnings("unchecked")
            List<NodeIOData> nodeOutputs = (List<NodeIOData>) value;
            inputs.addAll(new ArrayList<>(deepCopyList(nodeOutputs)));
        }
    }
}
```

## 6. 风险分析

### 6.1 技术风险
| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| NodeIOData序列化问题 | 低 | NodeIOData已实现Serializable，可安全放入Map |
| 内存占用增加 | 低 | 只在执行期间临时存储，工作流结束后释放 |
| 并发安全 | 低 | OverAllState.updateState()是线程安全的 |

### 6.2 业务风险
| 风险 | 等级 | 缓解措施 |
|------|------|----------|
| 现有工作流受影响 | 无 | 用户明确不考虑向后兼容 |
